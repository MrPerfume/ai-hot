执行环境如何“套上缰绳”而不是锁死它？

AI Agent 的能力来自对系统的深度操作权限，但权限开得越大，被攻击面就越宽。

npm 投毒、API Key 窃取、Agent 误操作导致系统故障——这些不再是假设场景，而是近期正在实际发生的事件。

研究报告显示，91% AI Agent 平台有漏洞，94% 可被投毒。

对此，众多大厂的安全团队都在密集发布 Agent 安全研究。

其中的关键问题在于，如何在安全性和可用性之间找到平衡点，给执行环境“套上缰绳”，而不是“锁死”它。
